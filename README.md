# Salah Times for macOS

A lightweight, native **menu-bar** prayer-times app for macOS. It lives in the
status bar (no Dock icon), counts down to the next prayer, and sends
notifications — with the same astronomical accuracy as the original
Java/Swing **[SalahTimes](https://github.com/Khoshimjonov/SalahTimes)** desktop
app it was ported from.

- **Platform:** macOS 14 (Sonoma) or later — Universal (Apple Silicon + Intel)
- **Built with:** SwiftUI + AppKit, with a standalone SwiftPM calculation core
- **Languages:** English · Oʻzbekcha · Русский

---

## Features

- **Menu-bar countdown** — the next prayer's name and time remaining sit right
  in the status bar; click for the full dropdown.
- **Full daily timetable** — Imsak, Fajr, Sunrise, Dhuhr, Asr, Maghrib, Isha,
  plus *midnight* and *last third of the night*. The current/next prayer is
  highlighted.
- **Hijri date & Islamic events** — the dropdown shows today's Hijri date;
  the core also knows Eids, holy nights, blessed and fasting days.
- **Qibla direction & distance** to the Kaʿaba from your coordinates.
- **14 calculation methods** (see table below), **2 Asr schools**
  (Shafiʿi / Hanafi), and **4 high-latitude rules**
  (None, Middle of Night, One-Seventh, Angle-Based).
- **Per-prayer adjustments** — fine-tune any prayer by ± minutes, and set the
  Imsak offset before Fajr.
- **Notifications** — get notified *on time* and/or a configurable number of
  minutes *before* each prayer.
- **Location** — enter coordinates, elevation and timezone manually, or type an
  address and hit **Look up** to geocode it (OpenStreetMap Nominatim +
  open-elevation).
- **Launch at login** and a customizable menu-bar label (toggle the prayer name,
  remaining time, an icon, short names, or per-second precision).
- **Fully localized** UI in English, Uzbek, and Russian.

## Installation

1. Download `SalahTimes.app.zip` from the
   [latest release](https://github.com/Khoshimjonov/SalahTimes_mac/releases/latest).
2. Unzip it and drag **SalahTimes.app** into `/Applications`.
3. The app is **ad-hoc signed, not notarized**, so on first launch macOS
   Gatekeeper will warn you. Either:
   - **Right-click** the app → **Open** → **Open** in the dialog, **or**
   - clear the quarantine flag from Terminal:
     ```sh
     xattr -dr com.apple.quarantine /Applications/SalahTimes.app
     ```
4. The app appears in the menu bar (it has no Dock icon). Open **Settings** from
   the dropdown to set your location and preferences.

## Building from source

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from `project.yml` — `SalahTimes.xcodeproj` is **not** checked in.

```sh
# Prerequisites: Xcode 15+, and XcodeGen (`brew install xcodegen`)

# 1. Generate the Xcode project from project.yml
xcodegen generate

# 2. Build the app
xcodebuild -project SalahTimes.xcodeproj -scheme SalahTimes \
           -configuration Release -destination 'platform=macOS' build

# 3. (Optional) Build a Universal binary
xcodebuild -project SalahTimes.xcodeproj -scheme SalahTimes \
           -configuration Release -destination 'generic/platform=macOS' \
           ARCHS="arm64 x86_64" ONLY_ACTIVE_ARCH=NO build
```

Run the pure calculation core's tests without Xcode:

```sh
cd SalahCore && swift test
```

> Whenever you change sources, resources, or build settings, edit `project.yml`
> and re-run `xcodegen generate`. Never hand-edit `project.pbxproj`.

## Architecture

The project is deliberately split into two layers:

```
SalahCore/        Standalone SwiftPM package — all the math.
                  Pure value types, Sendable, zero UI/app dependencies.
                  Prayer times, Hijri calendar, Islamic events, Qibla, astronomy.
                  Builds and tests without Xcode.

SalahTimes/       The Xcode app target (SwiftUI + AppKit).
  MenuBar/        Status-item controller, app state, label formatting.
  UI/             Dropdown popover and Settings window.
  Settings/       Observable settings model + JSON persistence.
  Location/       Async geocoder (Nominatim + open-elevation).
  Notifications/  Prayer notification scheduler (UserNotifications).
  Autostart/      Launch-at-login via SMAppService.
  Resources/      Assets + en/uz/ru localizations.

tools/            Java fixture generator and a Swift icon maker.
project.yml       XcodeGen source of truth for the Xcode project.
```

`PrayerTimesCalculator.swift` is a **line-by-line port** of the Java reference's
`SalahTimesCalculator.java`; its method structure mirrors the original so a
parity test can exercise every code path.

## Calculation parity

This app's whole reason to exist is to reproduce the math from the original
[SalahTimes](https://github.com/Khoshimjonov/SalahTimes) Java app **exactly**. The Swift calculator is verified to produce **second-identical**
output to the Java reference across **8064 checks** (896 fixtures × 9 times) —
currently **0-second drift** — covering every combination of coordinates,
elevation, timezone, method, Asr school, high-latitude rule, Imsak offset, and
date. Fixtures are generated by a self-contained Java program in
`tools/fixture-generator/` that copies the reference math verbatim.
